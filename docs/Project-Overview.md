# Project Overview

## Project Title

GCP IAM Security Lab: Implementing Least-Privilege Access Control Using Service Accounts and Custom IAM Roles

## Objective

The objective of this project was to implement and secure Identity and Access Management (IAM) controls within Google Cloud Platform (GCP). The project demonstrates how cloud administrators can manage identities, assign permissions, create custom roles, and enforce the Principle of Least Privilege (PoLP).

## Technologies Used

- Google Cloud Platform (GCP)
- Google Cloud IAM
- Service Accounts
- Compute Engine API
- Google Cloud Shell
- gcloud CLI

## Security Concepts Demonstrated

- Identity and Access Management (IAM)
- Role-Based Access Control (RBAC)
- Principle of Least Privilege (PoLP)
- Service Account Security
- Custom IAM Roles
- Cloud Governance
- Access Control Verification

## Project Outcomes

- Created and configured a GCP project
- Enabled required cloud services
- Created a dedicated service account
- Assigned least-privilege permissions
- Developed a custom IAM role
- Applied IAM policies using gcloud CLI
- Verified access controls through policy inspection
- Performed secure cleanup procedures

- Cloud Security
- IAM Administration
- GCP Security Management
- Access Control Implementation
- Security Automation
- Security Documentation
